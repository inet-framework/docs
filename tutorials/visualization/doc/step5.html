Step 5. Using 3D models for network nodes¶

Goals¶

Although the environment is similar to the real world, customizing network nodes may further ease the understanding of the simulation. In this step, we replace default icon of the nodes with an external 3D osg model.

The model¶

So far, we only deal with the environment of the simulation. In this step, we add network nodes to the simulation model. We add two WirelessHost, pedestrian0 and pedestrian1 and one AccessPoint, accessPoint0. The pedestrians will communicate with each other via accessPoint0 using a VoIP application. Communication will be configured in later steps.

The following picture shows what the network looks like and what the default appearance of the WirelessHost nodes looks like.

To use a 3D model instead of a 2D icon, set the osgModel parameter of the network node. Examine the following code snippet.

[Config Visualization05]
extends = Visualization04
description = Using 3D Models for Network Nodes
network = VisualizationD

# replacing cellphone icons with boxman models
*.pedestrian0.osgModel = "boxman.osgb.(0.05).scale"
*.pedestrian1.osgModel = "boxman.osgb.(0.05).scale"

The above osgModel parameter value will be explained below.

  • boxman.osgb is the name of the external 3D model,
  • (0.05).scale means that size of the external 3D model is decreased to 5%.

By default, appearance of the nodes is the same in 2D and 3D visualization. The default icon of WirelessHost is a laptop. In this tutorial, the WirelessHosts represent pedestrians. In 2D visualization, we have cellphone icon that represents pedestrians better, than a laptop icon. Icon can be replaced in the display string of the network node.

The following code snippet shows VisualizationD network which is used for this step.

network VisualizationD extends VisualizationC
{
    submodules:
        pedestrian0: WirelessHost {
            parameters:
                @display("p=398,89;i=device/cellphone");
        }
        pedestrian1: WirelessHost {
            parameters:
                @display("p=548,89;i=device/cellphone");
        }
        accessPoint0: AccessPoint {
            parameters:
                @display("p=487,163");
        }
        configurator: Ipv4NetworkConfigurator {
            parameters:
                @display("p=100,250");
        }
        radioMedium: Ieee80211ScalarRadioMedium {
            parameters:
                @display("p=100,350");
        }
}

An Ipv4NetworkConfigurator module instance and an Ieee80211ScalarRadioMedium module instance are also added to the model. We will use them as configurator and radioMedium. The configurator module assigns IP addresses and sets up static routing for an IPv4 network, radioMedium is a radio medium model, uses scalar transmission power in the analog representation.
